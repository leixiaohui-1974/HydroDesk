- 验证学校服务器是否可连通、可写入、可承载发布目录
- 选取一个最小样本目录完成探针上传
- 在验证成功后，再扩展到站点、文档、案例成果和源码镜像

### 3.3 直连测试结果

当前结果：

- **失败**
- 错误：`ssh: connect to host 27.188.65.247 port 4422: Network is unreachable`

后续进一步诊断后，结果更新为：

- `27.188.65.247:4422` 的 TCP 端口可达
- 但 SSH 握手在 `kex_exchange_identification` 阶段被远端关闭
- 客户端侧表现为 `end of file` / `Connection closed by remote host`

这说明：

- 问题不在 IP 或端口是否存在
- 而在当前执行环境的 SSH 直连路径不可用
- 不能稳定依赖 `27.188.65.247:4422` 作为当前发布通道

### 3.5 探针发布实测

使用通道：

```text
/Users/rainfields/lazycat_automount/hydrosis/research-publish/ppt-probe
```

已完成的真实发布验证：

- 创建远端目录 `research-publish/ppt-probe`
- 写入 `ppt_probe.txt`
- 上传单个 `pptx`
- 上传整个 `ppt/发布会26.4/output`

实测结果：

- 本地源目录文件数：`14`
- 本地源目录总大小：`597249096`
- 远端探针目录真实文件数：`31`
- 远端总大小：`597314658`

差异原因：

- 远端通过 `smbfs` 挂载
- macOS 会生成 `._*` AppleDouble 元数据文件
- 这些文件不是业务内容，但会增加远端文件数与少量体积

因此当前结论更新为：

- **学校服务器成果发布可行**
- **当前可用发布通道不是直连 SSH，而是懒猫微服挂载目录**

## 4. 结论

当前结论已经更新为：

- **直连 SSH 路径当前不可作为稳定发布手段**
- **但通过懒猫微服挂载路径，已经实现实际可写、可同步、可验证的远端发布**

因此后续的正式发布建议优先基于挂载通道推进，而不是继续强依赖直连 SSH。

## 5. 可能原因

优先级从高到低：

1. **当前网络环境无法访问外部该 IP/端口**
   - 本地网络策略限制
   - 当前网络未连学校 VPN
   - 宽带/校园网策略阻止该端口

2. **学校服务器端口对当前来源不可达**
   - SSH 端口变更
   - 防火墙未开放到当前公网来源
   - 仅允许校园网/VPN 内访问

3. **本地路由层问题**
   - 当前终端所在网络无外网路由
   - 本机网络状态异常

- 当前网络是否能访问学校内网/教育网资源
- 是否能从你自己的终端手工执行：
  - `ssh -p 4422 lei@27.188.65.247`

如果你本地手工也不通，那么问题不是发布脚本，而是网络通路本身。

### 阶段 B：如果后续要恢复直连 SSH，再做修复

当前不是必须先修复 SSH 才能发布，因为挂载通道已经可用。

但如果后续仍希望恢复标准 SSH 发布，应继续排查：

- 服务器侧 `sshd`

## 7. 正式发布架构建议

不建议将整个 `research/` 全量同步。

建议远端目录结构：

```text
<lazycat-mount>/research-publish/
  ├── apps/
  │   └── hydrodesk-web/
  ├── docs/
  │   └── hydrodesk/
  ├── artifacts/
  │   └── cases/
  ├── repos/
  │   └── research-src/
  └── probes/
      └── ppt-probe/
```

含义：

- `apps/hydrodesk-web`：放站点构建结果
- `docs/hydrodesk`：放文档、总结、路线图
- `artifacts/cases`：放案例成果物
- `repos/research-src`：放精简后的源码镜像
- `probes/ppt-probe`：放连接测试和上传探针目录

## 8. 发布策略建议

### 8.1 分层发布

建议拆成 4 条发布通道：

1. `web`
2. `docs`
3. `artifacts`
4. `src`

### 8.2 优先使用 rsync

正式发布建议优先用：

- `rsync 到挂载目录`

原因：

- 支持增量同步
- 支持排除规则
- 支持 dry-run
- 当前通路已验证真实可写
- 不再依赖当前不稳定的 SSH 握手链路

### 8.3 不要全量同步的目录

后续正式发布时，应默认排除：

- `.git/`
- `node_modules/`
- 各类 `dist/` 之外的构建缓存
- 日志目录
- 临时文件
- 私有配置
- 不必要的大原始数据
